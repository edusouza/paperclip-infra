# Resumo da sessão — 2026-05-06

## O que foi feito

Deploy completo do Paperclip (control plane de AI agents) no GCP + CloudFlare via OpenTofu.

## Stack final

- **VM:** GCE e2-standard-2 (2 vCPU, 8 GB) / Ubuntu 22.04 / us-central1-a
- **App:** Paperclip (Node.js, porta 3100, systemd)
- **DB:** PostgreSQL 16 local na VM
- **Exposição:** Cloudflare Tunnel → `paperclip.eduardosouza.dev` (sem Nginx, sem Certbot)
- **IaC:** OpenTofu com providers `google ~>5.0`, `cloudflare ~>4.0`, `random ~>3.6`

## Principais aprendizados

1. **CloudFlare provider v4** renomeou `cloudflare_tunnel` → `cloudflare_zero_trust_tunnel_cloudflared`
2. **`npx paperclipai onboard --yes` é proibido em servidor** — força modo local e ignora todas as env vars de deployment. Sempre rodar sem `--yes` interativamente
3. **`sudo -u paperclip`** requer `cd /home/paperclip` antes de qualquer comando npm
4. **APIs do GCP** devem ser ativadas antes do `tofu apply` (`compute.googleapis.com`, `iam.googleapis.com`)
5. **Onboarding não é automatizável** no startup.sh — feito manualmente via SSH após boot
